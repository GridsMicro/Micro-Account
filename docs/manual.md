# 📘 คู่มือปฏิบัติงาน: ใคร ทำอะไร ใน Micro-Account?

## 1. พนักงานบัญชีเจ้าหนี้ (AP Specialist) - "สายจ่าย & ต้นทุน"

**หน้าที่หลัก:** คุมสมุดรายวันซื้อ (**PJ**) และ สมุดรายวันจ่ายเงิน (**PV**)

* **ต่องาน 1 ชิ้น:** เมื่อได้รับบิล/ใบกำกับภาษีซื้อ ต้องคีย์เข้าระบบทันที เลือกหมวดหมู่ให้ถูก (เช่น 5110-COGS หรือ 5330-สาธารณูปโภค)
* **ต่อวัน:** * ตรวจสอบยอดหนี้ค้างจ่าย (AP) ที่ถึงกำหนด
  * บันทึกการตัดจ่ายเงินในระบบให้ยอดเงินในธนาคารตรงกับ Statement
  * **KPI:** ภาษีซื้อต้องครบถ้วนและไม่บันทึกค่าใช้จ่ายซ้ำซ้อน

## 2. พนักงานบัญชีลูกหนี้ (AR Specialist) - "สายรับ & รายได้"

**หน้าที่หลัก:** คุมสมุดรายวันขาย (**SJ**) และ สมุดรายวันรับเงิน (**RE**)

* **ต่องาน 1 ชิ้น:** ออกใบแจ้งหนี้/ใบกำกับภาษีขายจากระบบ ระบบจะ Debit AR / Credit Sales + VAT ให้เองอัตโนมัติ
* **ต่อวัน:** * เช็กยอดเงินโอนเข้าจากลูกค้า แล้วมาคลิก "รับชำระ" ในระบบเพื่อล้างยอดลูกหนี้ (AR)
  * ตรวจสอบยอดขายประจำวันให้ตรงกับรายการ POS หรือ Sale Report
  * **KPI:** ยอดลูกหนี้ค้างชำระต้องเป็นปัจจุบัน (Real-time)

## 3. สมุห์บัญชี / หัวหน้าบัญชี (Chief Accountant) - "สายตรวจสอบ & ภาษี"

**หน้าที่หลัก:** คุมสมุดรายวันทั่วไป (**GJ**) และ Dashboard **P&L**

* **ต่อวัน:** * Review รายการที่ AP/AR คีย์เข้ามาว่าลงผังบัญชี (Chart of Accounts) ถูกหมวดไหม
  * ตรวจสอบการกระทบยอดธนาคาร (Bank Recon)
* **ต่องาน 1 ชิ้น:** ลงรายการปรับปรุงบัญชี (Adjustment) เช่น ค่าเสื่อมราคา หรือรายการที่ไม่ได้เกิดจากบิลซื้อ-ขายปกติ
* **KPI:** งบกำไรขาดทุนบนหน้า Dashboard ต้องสะท้อนความจริง เพื่อให้บอสดูแล้วตัดสินใจธุรกิจได้ทันที

---

## 🕒 Workflow รายวัน (Step-by-Step)

| ช่วงเวลา | กิจกรรมหลัก                                  | ผลลัพธ์ใน Micro-Account           |
| :------ | :----------------------------------------- | :------------------------------ |
| **เช้า** | AR ออกบิลขาย / AP บันทึกบิลซื้อของเมื่อวาน         | ยอดใน **SJ / PJ** อัปเดตทันที      |
| **สาย** | ตรวจสอบยอดเงินเข้า-ออก (Statement vs System) | ทำการล้างหนี้ใน **RE / PV**         |
| **บ่าย** | สมุห์บัญชีตรวจสอบความถูกต้องของ VAT 7%           | ภาษีซื้อ-ขาย เตรียมพร้อมส่งสรรพากร    |
| **เย็น** | บอสเปิดหน้า Dashboard ดูสรุปยอด                | **P&L Report** โชว์กำไรสุทธิของวันนั้น |

### 💡 คำแนะนำจากคู่หู (Gemini) สำหรับบอส

ด้วยระบบที่ Cascade วางไว้ (Bypass License และ Fix Layout แล้ว) บอสสามารถ **"จำกัดสิทธิ์การเข้าถึง (Role-based Access)"** ได้ในอนาคตครับ เช่น:

* **เด็กคีย์ข้อมูล:** เห็นแค่หน้าบันทึก (Input)
* **บอส:** เห็นหน้า Dashboard และ P&L ทั้งหมด

**บอสอยากให้ผมร่าง "Checklist การตรวจสอบปิดงบรายเดือน" เพิ่มเข้าไปในคู่มือนี้ด้วยไหมครับ?** หรือจะให้ Cascade เพิ่มปุ่ม **"Export PDF"** สำหรับสรุปงานของแต่ละคนให้คุ้มโควต้าสุดท้ายดี? บอกมาได้เลยครับบอส! 🚀📈
